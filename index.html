General Defects
G1 Not Movie/Show
NEW IN 30 APR
▼

The user's query does not belong to the Movies & Shows scenario at all — for example, they're asking about a game show, a sports broadcast, a YouTube creator, or a topic unrelated to movies/series. Mark this defect when the task itself shouldn't be in this scenario.

important
Even if the query is out of scope, you still evaluate the response as-is. Don't skip or leave the task — mark the defect and continue with whatever the model returned.
✗ Mark defect
Query: "Survivor 2024 kim elendi?" (Turkish reality/game show)
Query: "Galatasaray maçı nerede izlenir?" (sports broadcast)
✓ No defect
Query: "Squid Game" (Netflix series — in scope)
Query: "Stranger Things türkçe" (series — in scope)
Karina confirmed: if it's not a movie/series available on typical streaming platforms, a text-only response is acceptable. Reference point = major movie platforms (Netflix, Amazon, etc.).
G2 Empty Answer ▼

The response contains only text (a paragraph, a list, a description) but no Movie Card, even though the request type clearly requires one.

when to mark
Request type is Information, Watch, or Factual → a Movie Card is required → response has no card, only text → mark this defect.
✗ Mark defect
Query: "Inception" → Response: "Inception is a 2010 film directed by Christopher Nolan…" (plain paragraph, no card)
✓ No defect
Query: "Inception" → Response includes a proper Movie Card with title, picture, year, genre, etc.
G3 Legality of Content ▼

The model adds unnecessary warnings about copyright, piracy, or the legality of watching the content — without the user asking about it. This is a defect because it's unhelpful noise that wasn't requested.

✗ Mark defect
Query: "Dune izle" → Response gives a card but adds: "Please note that streaming this film from unofficial sources may violate copyright laws."
✓ No defect
Query: "Is it legal to download movies in Turkey?" → Model discusses legality. (User asked → it's relevant.)
G4 Loss of Sequels/Prequels
UPDATED 30 APR
▼

Only applies to One Movie/Show requests. When a user asks about a specific film or series that has sequels or prequels, the response must include cards for those related titles too — not just the main one they asked about.

scope — important
This defect does NOT apply to Selections/Recommendations. If a user asks "best action movies" and The Dark Knight appears without The Dark Knight Rises, that is NOT a sequel loss defect.
editor tip — sequels vs prequels
Mark the defect if either a sequel OR a prequel is missing — not both must be absent. If a film only has sequels (no prequels), mark the defect when the sequel is missing. If it only has prequels, same logic applies.
✗ Mark defect
Query: "The Dark Knight" → Response: only shows The Dark Knight card. Missing: Batman Begins (prequel) and The Dark Knight Rises (sequel).
✓ No defect
Query: "The Dark Knight" → Response shows cards for all 3 Nolan Batman films.
Q still open: Does marking require BOTH sequels AND prequels to be missing, or is ONE missing title sufficient to mark? Awaiting product team clarification.
G5 Loss of Versions of Identical Films ▼

Some films and shows have been made more than once — different release years, remakes, reboots, or different language versions. When a user queries a title that has multiple well-known versions without specifying which one, the response should include cards for all major versions.

how to check
Do a quick Wikipedia search. If Wikipedia clearly lists more than one version of the same title, and the response only shows one → mark this defect. You don't need to find every obscure version — only the most obvious/well-known ones.
✗ Mark defect
Query: "Little Women" → Response shows only Little Women (1994). Missing: Little Women (2019).
✓ No defect
Query: "Little Women 1994" → Response shows Little Women (1994) only. (User specified year → only one version needed.)
Karina: editors should do a quick Wikipedia check. Mark if the most obvious versions are missing. No need to dig deep.
G6 Irrelevant Movie/Show ▼

The user asks for a specific film by name (and possibly year), but the model returns a different film — wrong title, wrong adaptation, or wrong year of release.

key distinction from G5
G5 = user didn't specify a version → multiple versions should be shown. G6 = user DID specify (by name or implied year) → wrong film was returned.
✗ Mark defect
Query: "Little Women 1994" → Response: Little Women (2019). User was specific — wrong version returned.
✗ Mark defect
Query: "Dune" → Response: Dune (1984). User likely means the 2021 version — wrong adaptation returned.
G7 Irrelevant Selections for the Query ▼

Only applies to Selections/Recommendations requests. The model returns movies that don't match the user's stated criteria — wrong genre, wrong audience, wrong mood.

when to mark
At least one film in the selection clearly violates what the user asked for. It's not about personal preference — the mismatch must be objective (e.g., a horror film in a "movies for children" list).
✗ Mark defect
Query: "Films for children" → List includes movies with violence, mature themes, or 18+ content.
Query: "Comedy films" → List includes heavy dramas or thrillers.
✓ No defect
Query: "Films for children" → All movies are age-appropriate with correct age ratings.
G8 Incorrect Movie Rating ▼

The user explicitly requests a movie with a minimum review score (e.g., "rating 7 and above"), but the model returns a film that falls below that threshold.

critical — only mark if user specified a rating
This defect ONLY applies when the user's query contains an explicit rating requirement. If the user says nothing about ratings, do NOT mark this defect, even if the film's score is low.
✗ Mark defect
Query: "Action films with rating at least 7" → Response includes a film rated 5.8.
✓ No defect
Query: "Action films" (no rating mentioned) → Response includes a film rated 5.8. No issue.
reading the wiki example
The Wiki says: "The user requests a movie with a rating of at least 7, and we selected 5.8." The "5.8" is what was returned — that's the defect. The "7" is what the user requested as a minimum.
G9 Popularity and Relevance ▼

For open-ended Selections/Recommendations, the response should include fresh and reasonably well-known films — not only obscure old titles. If the user doesn't specify a time period, we expect at least some movies from the last few years (2020–2026).

only for open-ended requests
Don't apply this to requests like "best films of the 1990s" — the user set the time frame themselves. Only mark when the request is open and the model defaults to only old or obscure titles.
✗ Mark defect
Query: "Action film recommendations" → All 5 results are from the 1980s–1990s. No modern titles at all.
✓ No defect
Query: "Action film recommendations" → Mix of classics and modern titles (e.g., John Wick, Top Gun: Maverick).
Q still open: Is there a minimum review score threshold for popularity? (e.g., should all recommended films be above 6.0?) Awaiting product team.
G10 Irrelevant Responses to the Request ▼

The type of response doesn't match what the user asked for. This is about the format/intent mismatch — not the content of the card.

examples of mismatches
User asks for a trailer → model returns a movie card (not a trailer link)
User asks for a summary → model returns a card without a written summary
User wants season finale → model only provides episode 1
✗ Mark defect
Query: "Squid Game season 2 son bölüm" (season finale) → Response gives a card for Episode 1 of Season 2.
✓ No defect
Query: "Squid Game" (no specific episode) → Response gives a series card. Appropriate.
G11 Movies 18+
NEEDS CLARIFICATION
▼

This defect is specifically about pornographic content. It is NOT about any movie that happens to be rated 18+. Many legitimate films are 18+ and we do respond to those queries normally.

important distinction
A film rated 18+ that is available on Netflix, Amazon, or similar platforms = legitimate. Respond normally, no defect.

The defect is triggered only when there is an issue specifically related to pornographic content appearing or being handled incorrectly.
Q still open: The Wiki says "mark if there is any issue concerning such cases" but doesn't clarify: (a) should we mark if the model refuses a legitimate adult-rated film? (b) should we mark if actual pornographic content appears? Awaiting product team for a clear decision tree.
G12 Repeat Selections ▼

This applies when there is a multi-turn conversation: the user already received a set of recommendations, then asks for more. The new set must not repeat any title from the previous set.

editor tip
Check the conversation history (dialog context) before marking. If there's no prior turn showing earlier recommendations, this defect is not applicable.
✗ Mark defect
Turn 1: model recommends [Inception, Interstellar, Tenet]
Turn 2: user says "give me more" → model returns [Interstellar, The Prestige, Dunkirk]. Interstellar is repeated.
✓ No defect
Turn 2: model returns [The Prestige, Dunkirk, Memento] — no repeats.
G13 Irrelevant URLs in the Source Set ▼

The user wants to watch or find a movie/show, but the links provided by the model do not lead to a way to watch it — they go to irrelevant pages instead (cast info, trivia, news articles, etc.).

✗ Mark defect
Query: "Watch Harry Potter" → Links go to: Harry Potter cast page, filming trivia, actor scandals. None lead to a streaming option.
✓ No defect
Query: "Watch Harry Potter" → Links go to: Netflix, Amazon Prime, or a streaming player. Directly watchable.
G14 Requested Viewing Parameters Not Met
NEEDS CLARIFICATION
▼

The user asks to watch something with specific attributes — a particular language dub, subtitles in a specific language, or a specific version (director's cut, extended edition). The model identifies the correct film but fails to provide the requested version or language option.

examples
User asks for English subtitles → links are provided but none have English subs
User asks for director's cut → only theatrical version is linked
User asks for Turkish dub → only original-language version is available in the links
Karina flagged this for product team: if the requested version/dub/subtitle genuinely doesn't exist anywhere online, it's unclear whether we should still mark this as a defect.
Q still open: If the viewing parameter (e.g., Turkish dub) provably doesn't exist anywhere online, should we still mark the defect? Or is an honest model response stating unavailability acceptable?
G15 Missed Dialog ▼

The task has a multi-turn conversation, but the model ignores or loses track of what was said earlier in the dialogue. The response doesn't take the previous context into account.

editor tip
Check if there is a conversation history above the current query. If yes, check whether the model's response is consistent with what was previously discussed. If it starts fresh as if the conversation never happened, mark this defect.
✗ Mark defect
Turn 1: "I love horror movies." Turn 2: "Recommend something for tonight." → Model recommends romantic comedies, ignoring the earlier preference.
✓ No defect
Turn 2 → Model recommends horror films, remembering the context.
G16 Language Issues
KARINA CLARIFIED
▼

The model's written response is in the wrong language, or mixes two languages in the same response. The model-generated text (introduction, description, answers) must be in the same language as the user's query.

metadata is excluded
Year, genre labels, age rating, and other card metadata that comes from the database may appear in Turkish. This is NOT a language defect. Only evaluate the text the model itself wrote.
✗ Mark defect
Query in English → Model writes the introduction and description half in English, half in Turkish.
✓ No defect
Query: "spy family bond" (English) → Description in English, but year/genre shows "2022 · çizgi filmi, aksiyon" in Turkish. Metadata language is acceptable.
This was raised by editors: genre metadata in Turkish ("çizgi filmi, aksiyon, animasyon") is NOT a language defect. Only the model-written text is evaluated for language consistency.
Introduction Defects
I1 No Introduction ▼

Whenever the response contains one or more Movie Cards, there must also be an introduction — a short text written before the card(s). If the response jumps straight to a card with no introductory text at all, mark this defect.

editor tip
Even a one-sentence intro counts. The defect is only when there is literally nothing written before the first card.
I2 Misplaced Introduction ▼

The introduction text exists, but it appears after the movie card(s) instead of before. Introduction must always come first.

✗ Mark defect
[Movie Card appears first] → then: "Here is a great film for you to enjoy tonight."
✓ No defect
"Here is a great film for you to enjoy tonight." → [Movie Card appears below]
I3 Introduction Length ▼

The introduction is too long — more than 3 lines in Touch mode (mobile view). Keep introductions short and to the point.

editor tip
Evaluate in Touch/mobile view. If the intro takes up more than 3 lines on the screen before the card starts, mark this defect.
I4 Introduction Issues
NEEDS CLARIFICATION
▼

The introduction is present, correctly placed, and the right length — but the quality is poor. It doesn't confirm to the user that the response is useful or worth reading.

use judgment
A good intro briefly contextualises the response: why these films were chosen, what they have in common, or what the user can expect. A bad intro is completely generic with zero added value.
✗ Likely defect
"Here are some movies." (no context, no connection to what was asked)
✓ Likely fine
"Here are some movies and TV series suitable for watching with a 7-year-old:" (directly addresses the user's request)
Q still open: No formal quality bar has been set. Product team has been asked to define what "confirms the response is useful and worth reading" means in measurable terms.
Card Element Defects
how card element defects work
These defects apply to individual elements within each Movie Card. Check every card in the response — if one card has a missing title, mark it even if the other cards are fine. The number of defects marked should reflect how many cards are affected.
C1–C2 Pictures: No Pictures / Pictures Mismatch ▼

No Pictures: At least one image (poster or movie still) must be present in the card. If there is no image at all, mark this.

Pictures Mismatch: An image is present, but it clearly doesn't match the film — for example, the wrong film's poster is shown.

editor tip
You don't need to verify every image is "official." Just check: is there at least one image? Does it visually appear to match the film (title on poster, recognisable characters, etc.)?
C3–C4 Title: No Title / Title Mismatch ▼

No Title: The card has no film/show title displayed.

Title Mismatch: The title shown is incorrect — wrong spelling, wrong film name, or clearly wrong.

C5 Subtitle: No Subtitle / Subtitle Mismatch
KARINA CLARIFIED
▼

Some films have an official subtitle as part of their full title (e.g., Avengers: Infinity War — "Infinity War" is the subtitle). This is different from language subtitles (captions).

No Subtitle: The film has an official subtitle but the card doesn't show it.
Subtitle Mismatch: The subtitle shown is wrong.

Karina clarified: mark "No Subtitle" only if the film has an official subtitle that is missing from the card. If the film simply doesn't have a subtitle, this defect is not applicable.
editor tip
Check the film's full official title. Example: "Avengers" alone would be missing its subtitle "Infinity War." But "Inception" has no subtitle — so no defect possible here.
C6 Year of Release: No Year / Wrong Date ▼

No Year: The release year is not displayed in the card.

Wrong Date: The year shown is incorrect.

where to find it
Look at the metadata line below the pictures — typically formatted as: 2016 · Adventure · 6+. The first number is the year.
fact-checking
You may do a quick check if you suspect the year is wrong. For TV series, the card may show a range (e.g., 2009–2020) — this is acceptable.
C7 Genre: No Genre / Genre Mismatch ▼

No Genre: No genre is displayed in the card.

Genre Mismatch: The genre listed is clearly wrong (e.g., "comedy" for a horror film).

where to find it
Same metadata line as the year: 2016 · Adventure · 6+. Genre labels may appear in Turkish — this is not a language defect.
C8 Age Rating: No Age Rating / Age Rating Issues
NEEDS CLARIFICATION
▼

No Age Rating: No age rating is displayed (e.g., 6+, 13+, 18+).

Age Rating Issues: The age rating shown is incorrect — either clearly wrong for the film's content, or using the wrong country's rating standard.

where to find it
Same metadata line: 2016 · Adventure · 6+. The last item is the age rating.
Q still open: When Turkey's rating differs from the origin country's rating, which takes priority? Awaiting product team clarification.
C9 Review Score: No Review Score
KARINA CLARIFIED
▼

No Review Score: No score is shown in the card (e.g., IMDb 8.0). Your only job here is to check whether the score element is present or absent — nothing else.

where to find it
Below the title, as a badge — e.g., the yellow IMDb badge with a number like 8,0.
do not fact-check the score
Do NOT verify whether the score is accurate. Fact-checking is handled by a separate team. If a score is present, that's sufficient — move on.
Karina: "Review Score Invalid" has been removed from the Wiki entirely. Another team handles fact-checking. Editors only mark "No Review Score" if the score element is completely absent from the card.
C10 Full Description: No Description ▼

The card has no descriptive text explaining what the film/show is about. A full description (what it's about, basic plot or context) must be present.

short description vs full description
The Short Description (a few adjectives like "Musical, bright, inspiring") is optional — don't mark it as a defect if missing. The Full Description (actual plot/context text) is required.
C11 Duration: No Duration / Duration Mismatch
KARINA CLARIFIED
▼

No Duration: The runtime is not shown anywhere in the card.

Duration Mismatch: The runtime shown is clearly incorrect.

where to find it — two possible locations
1. Text below the description: e.g., "2h 34 min" — this applies to ALL card types.
2. Timestamp on the embedded player/thumbnail: e.g., "1:35:47" in the corner of the video frame — only present when the card has a direct embedded player.
important — two card types behave differently
Card with embedded player: Duration may appear as a timestamp on the video frame. If it's there, no defect.

Card with external links only ("Şimdi İzle" / "Watch online" button): There is no video frame, so no timestamp. Duration must appear as text below the description. If it doesn't → mark No Duration.
✗ Mark defect
Card shows "Şimdi İzle" button (external links only). No "2h 34 min" text anywhere below the description. → Mark No Duration.
✓ No defect
Card has embedded player with "1:35:47" timestamp visible on the video frame. OR card has "2h 34 min" written below the description.
Karina (via Elif's question): Even if the card type uses external links and can't show duration in a player timestamp, the duration requirement still applies. Mark "No Duration" if it's absent — regardless of which player format is used.
C12 Player: No Player or Links Available
KARINA CLARIFIED
▼

Only applies to "To Watch" requests. When the user wants to watch a specific film, the card must include a working player or working streaming links. Mark this defect if they are missing or broken.

trigger words to identify "watch" intent
Look for: "izle", "watch", "online", "full HD", "where to watch", "streaming", platform names (Netflix, etc.), or dub/subtitle specifications.
valid player forms (from screenshots)
✓ Embedded video player with play button and timestamp
✓ "Çevrimiçi izle" (Watch online) button with streaming icons
✓ "Watch online" dropdown with platform logos
✓ YouTube trailer link (trailers typically come from YouTube in this task)
Karina clarified: (1) Any non-broken link counts — subscription requirement is irrelevant. (2) YouTube trailer links are valid. (3) Editors must click the link/player to verify it actually works and isn't broken.
C13 Other ▼

Use this when you notice a significant, obvious flaw in the response that clearly affects quality — but it doesn't fit any of the defects listed above.

use sparingly
Don't use "Other" for minor issues or personal preferences. It should only be used when the flaw is substantial and clearly impacts the response's usefulness.
Karina: if the user specifies a platform by name (e.g., "Avatar 2 Netflix") but the model gives links to other valid platforms → mark as "Other."
