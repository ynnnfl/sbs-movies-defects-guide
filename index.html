1 · General
G1 Not Movie/Show
NEW IN 30 APR
▼

The user's query does not belong to the Movies & Shows scenario at all — e.g., a game show, sports broadcast, YouTube creator, or anything that isn't a movie/series on a streaming platform. Mark this when the task itself shouldn't be in this scenario.

important
Even if the query is out of scope, still evaluate the response as-is. Don't skip the task — mark the defect and continue.
✗ Mark defect
Query: "Survivor 2024 kim elendi?" (Turkish game show)
Query: "Galatasaray maçı nerede izlenir?" (sports broadcast)
✓ No defect
Query: "Squid Game" (Netflix series — in scope)
Query: "Stranger Things türkçe" (series — in scope)
Karina: if it's not a movie/series on typical streaming platforms, a text-only response is acceptable. Major streaming platforms are the reference point.
G2 Empty Answer
KARINA CLARIFIEDUPDATED 5 MAY
▼

The response contains only text but no Movie Card, even though the request type requires one.

when to mark
Request is Information, Watch, or Factual → a card is required → response has only text, no card → mark this defect.
do element defects still apply?
Yes — if there is something to mark in General defects, mark it. Karina: "We'd like to analyse the responses according to general defects, if there is any." So even when "Empty Answer" is marked, continue evaluating the response for other applicable General defects. Element defects (No Pictures, No Genre, etc.) only apply if a card is partially present.
✗ Mark defect
Query: "Inception" → Response: a plain paragraph. No card.

Still check: are there other General defects applicable (e.g., Language Issues, Legality of Content)? Mark those too if present.
✓ No defect
Query: "Inception" → Response includes a proper Movie Card. Evaluate its elements normally.
G3 Legality of Content ▼

The model adds unnecessary copyright or piracy warnings without the user asking about it.

✗ Mark defect
Query: "Dune izle" → Card is returned but model adds: "Streaming from unofficial sources may violate copyright laws."
✓ No defect
Query: "Is it legal to download movies in Turkey?" → Model discusses legality. (User asked — it's relevant.)
G4 Loss of Sequels/Prequels
UPDATED 30 APRUPDATED 5 MAY
▼

Only applies to One Movie/Show requests. When a user asks about a specific title that has sequels or prequels, the response must also include cards for those related titles.

scope — important
Does NOT apply to Selections/Recommendations. If the user asks "best action movies" and The Dark Knight appears without its sequels, that is NOT this defect.
when to mark
If the film has both sequels AND prequels, both must be present. Mark this defect only when the response is missing at least one that should be there.
✗ Mark defect
Query: "The Dark Knight" → Only The Dark Knight card shown. Missing: Batman Begins (prequel) + The Dark Knight Rises (sequel).
✓ No defect
Query: "The Dark Knight" → All 3 Nolan Batman films shown.
Q9 still open: If sequels/prequels are mentioned in the text of the response but not shown as separate Movie Cards — is that still a defect? Or is a textual mention sufficient? Awaiting product team.
G5 The Loss of Versions of Identical Films
NEEDS CLARIFICATION
▼

Some films exist in multiple well-known versions (remakes, different release years). When a user queries a title without specifying which version, the response should include cards for all major versions.

how to check
Quick Wikipedia search. If Wikipedia clearly lists more than one well-known version and the response shows only one → mark this defect.
✗ Mark defect
Query: "Little Women" → Only Little Women (1994) shown. Missing: Little Women (2019).
✓ No defect
Query: "Little Women 1994" → Only 1994 version shown. User specified — no issue.
Karina: do a quick Wikipedia check. Mark if the most obvious/well-known versions are missing. No need to dig deep.
Q (versions as text vs card) still open: If the different versions are mentioned in the response text but not shown as separate Movie Cards — is that still a defect? Or is a textual mention sufficient? Awaiting product team.
G6 Irrelevant Movie/Show ▼

The user asks for a specific film by name (and possibly year), but the model returns a different film.

key distinction from G5
G5 = user didn't specify → multiple versions should be shown.
G6 = user DID specify → wrong film was returned.
✗ Mark defect
Query: "Little Women 1994" → Response: Little Women (2019).
Query: "Dune" → Response: Dune (1984). User likely means 2021.
✓ No defect
Query: "Dune 2021" → Response: Dune (2021). Correct.
G7 Irrelevant Selections for the Query ▼

Only for Selections/Recommendations. The model returns films that don't match the user's stated criteria — wrong genre, wrong audience, wrong mood.

when to mark
At least one film clearly and objectively violates what the user asked for.
✗ Mark defect
Query: "Films for children" → List includes movies with violence or 18+ content.
✓ No defect
Query: "Films for children" → All films are age-appropriate.
G8 Incorrect Movie Rating ▼

The user explicitly requests a minimum review score (e.g., "rating 7 and above"), but the model returns a film below that threshold.

only mark if user specified a rating
If no rating is mentioned in the query, do NOT mark this — even if the film's score is low.
reading the Wiki example
"User requests rating at least 7, we selected 5.8." → 7 = user's minimum. 5.8 = what was returned. That's the defect.
✗ Mark defect
Query: "Action films with rating at least 7" → Response includes a film rated 5.8.
✓ No defect
Query: "Action films" (no rating mentioned) → Film rated 5.8 appears. Fine.
G9 Popularity and Relevance
NEEDS CLARIFICATION
▼

For open-ended Selections, the response should include fresh and reasonably well-known films. If no time period is specified, expect at least some movies from recent years (2020–2026).

only for open-ended requests
Don't apply to requests like "best films of the 1990s." Only mark when the request is open and the model defaults to only old or obscure titles.
✗ Mark defect
Query: "Action film recommendations" → All 5 results are from the 1980s–1990s. No modern titles.
✓ No defect
Query: "Action film recommendations" → Mix of classics and modern titles (e.g., John Wick, Top Gun: Maverick).
Q3 still open: (a) Is there a minimum review score threshold for recommended films? (b) For queries that inherently invite classic/timeless recommendations — e.g., "movies that changed your perspective on life" (hayata bakış açınızı değiştiren filmler) — is it acceptable if the list consists mostly of well-known classics (Forrest Gump, The Matrix) with no recent titles? Or does the "fresh" rule still apply regardless of query type? Awaiting product team.
G10 Irrelevant Responses to the Request
UPDATED 5 MAY
▼

The response is completely off-target — wrong intent, doesn't address what the user asked at all.

key distinction — G10 vs G14
G10 (Irrelevant Responses): The response completely misses the point — wrong type of content, wrong intent entirely. The model didn't understand or address what the user asked for.

G14 (Viewing Parameters Not Met): The response is correct and on-point, but something specific is missing or doesn't match the user's request. Think of the coffee analogy in G14 — right coffee, wrong specs.
✗ Mark defect
Query: "Squid Game season 2 son bölüm" (finale) → Response gives Episode 1.
Query: "Inception trailer" → Full movie card returned, no trailer link.
✓ No defect
Query: "Squid Game" (no specific episode) → Series card returned. Appropriate.
G11 Movies 18+
KARINA CLARIFIEDUPDATED 5 MAY
▼

This defect covers two distinct cases related to adult content. It is not about any film that simply happens to be rated 18+.

case (a) — model refuses a legitimate adult film query
If a user asks about a regular adult-rated film available on major streaming platforms, and the model returns nothing (empty response, refusal, or irrelevant deflection) → mark this defect. Legitimate adult films should be responded to normally.
case (b) — model returns actual pornographic content
If the model returns pornographic content → mark this defect.
important distinction
A film rated 18+ on Netflix, Amazon, etc. = legitimate. Respond normally.
Pornography = not a legitimate streaming platform film. Always a defect.
Karina: We don't show pornography → always mark (b). For (a): if the model shows nothing to a regular adult-film query, that's also a defect. A film may sometimes seem like pornography but is a regular movie available on platforms — respond to those normally.
G12 Repeat Selections ▼

In a multi-turn conversation: the user got a first set of recommendations, then asked for more. The new set must not repeat any title from the previous set.

editor tip
Check the previous dialogue first. If there's no prior turn with earlier recommendations, this defect is not applicable.
✗ Mark defect
Turn 1: [Inception, Interstellar, Tenet]
Turn 2: "give me more" → [Interstellar, The Prestige, Dunkirk]. Interstellar repeated.
✓ No defect
Turn 2: [The Prestige, Dunkirk, Memento] — no repeats.
G13 Irrelevant URLs in the Source Set ▼

The user wants to watch a film, but the links don't lead anywhere watchable — they go to cast pages, trivia, news articles, etc. instead of streaming options.

✗ Mark defect
Query: "Watch Harry Potter" → Links go to: cast page, filming trivia, actor news. No streaming link.
✓ No defect
Query: "Watch Harry Potter" → Links go to Netflix, Amazon Prime, or a working player.
G14 Requested Viewing Parameters Not Met
KARINA CLARIFIEDNEEDS CLARIFICATIONUPDATED 5 MAY
▼

The user asks to watch something with specific viewing attributes. The model finds the correct film but fails to provide the requested version or language.

what are "viewing attributes"?
Think of it like ordering coffee: the film = the coffee (Americano). Viewing attributes = the specifications (iced/hot, less sugar, oat milk). If the wrong coffee comes → G10. If the coffee is right but served hot when you asked for iced → G14.

Viewing attributes include: dubbing language, subtitle language, specific version (director's cut, extended edition), named platform (e.g., "Avatar 2 Netflix").
examples
User asks for Turkish dub → only original-language version linked
User asks for English subtitles → links provided but none have English subs
User asks for director's cut → only theatrical version linked
service type (e.g. "free to watch", "without subscription")
Karina: use common sense for these cases for now. Further discussion pending.
Q14 still open: (a) If the requested parameter provably doesn't exist anywhere online, should we still mark this defect — or is an honest model response stating unavailability acceptable? (b) Does the editor need to do external research to verify whether the requested parameter actually exists online before deciding? Awaiting product team.
G15 Missed Dialog ▼

The task has a multi-turn conversation, but the model ignores or loses track of what was said earlier.

editor tip
Check if there's a previous dialogue above the current query. If the model's response is inconsistent with earlier context — or starts fresh as if no prior turns exist — mark this defect.
✗ Mark defect
Turn 1: "I love horror movies."
Turn 2: "Recommend something for tonight." → Model recommends romantic comedies.
✓ No defect
Turn 2 → Model recommends horror films, remembering the earlier context.
G16 Language Issues
KARINA CLARIFIEDNEEDS CLARIFICATIONUPDATED 5 MAY
▼

The model's written response is in the wrong language or mixes languages. The model-generated text (introduction, description) must match the language of the user's query.

ambiguous query language → always prioritise Turkish
If the query is a word that exists in both Turkish and English (e.g., "film"), or the language is otherwise ambiguous — the expected response language is Turkish. If a model responds in English for such a query, mark Language Issues.
✗ Mark defect
Query in English → Introduction and description are mixed English/Turkish.
Query: "film" (ambiguous) → Model responds in English. Turkish was expected.
✓ No defect
Query: "spy family bond" (English) → Description in English, "2022 · çizgi filmi, aksiyon" in Turkish. Fine.
Query: "film" (ambiguous) → Model responds in Turkish. Correct.
editor tip — what counts as model-written text
Metadata fields (year, genre, age rating) pulled from the database may appear in Turkish even when the query is in another language. This follows the same principle as Turkish labels in the E-com task. Only the text the model itself generates (introduction, description) is evaluated for language consistency.
Karina: if the query is ambiguous in terms of language, it should always be Turkish.
Q19 still open: Is it formally acceptable for card metadata fields (year, genre, age rating label) to appear in Turkish when the user's query is in another language? Awaiting product team.
2 · Elements
how element defects work
These defects apply to individual elements within each Movie Card. Check every card in the response — if one card has a missing element, mark it even if the other cards are fine.
E1 No Introduction
KARINA CLARIFIEDUPDATED 5 MAY
▼

Whenever the response contains one or more Movie Cards, there must be an introduction — a short text written before the card(s). Mark this if the response jumps straight to a card with nothing written before it.

two types of introduction — both count
In a Selection response, there can be two levels of introduction:

1. General selection intro — appears before all cards. Explains how the model understood the query and why these films were chosen.
2. Per-movie intro — appears before each individual card. A short annotation about that specific film.

Both are treated as "introduction" and both are subject to the 3-line max rule (E3). Mark "No Introduction" if either is missing where expected.
Karina: "The first part is the general introduction about the selection. Then we see the introduction for each movie in the selection. Both are introduction so we treat both as introduction which can't be longer than 3 lines."
E2 Introduction Misplaced ▼

The introduction exists but appears after the movie card(s) instead of before. Introduction must always come first.

✗ Mark defect
[Movie Card first] → then: "Here is a great film for you."
✓ No defect
"Here is a great film for you." → [Movie Card below]
E3 Introduction Length
KARINA CLARIFIEDUPDATED 5 MAY
▼

Any introduction — whether the general selection intro or the per-movie intro — must not exceed 3 lines in Touch mode (mobile view). Mark this if either exceeds the limit.

editor tip
Evaluate each introduction separately in Touch/mobile view. The 3-line limit applies to both: the general intro at the top AND each individual per-movie intro before each card.
Karina: both the general selection introduction and the per-movie introduction are subject to the same 3-line maximum rule.
E4 Introduction Issues
KARINA CLARIFIEDUPDATED 5 MAY
▼

The introduction is present, correctly placed, and the right length — but the quality is poor. It must contain substantive context justifying why the film(s) suit the user's request, or serve as a short annotation for the movie.

what counts as substantive
The intro should briefly explain: how the model understood the query, what the films have in common, or why they are relevant to the user. For Selections, "Here are some [genre] films:" is acceptable as a general opener — but the per-movie intros should add actual context about each film.
✗ Mark defect
"Here are some movies." (no context, no connection to what was asked)
"Here are my recommendations." (generic, adds nothing)
✓ No defect
Query: "action film recommendations" → "If you're looking for action, here are options that are both fast-paced and diverse. The following combine action tempo with different genres."
Per-movie intro: "İyimserlik ve hayatın tadı — Hayatın küçük anlarını büyüten, 'her şey bir yerde bir anlam bulur' hissi veren bir yolculuk." (short annotation describing the film's feeling)
Karina: there should be substantive context justifying that the movie/show suits the user or serves as a short annotation. For a selection, "here are some [x] films:" as a general opener is OK. Per-movie intros should add real context.
E5 No Pictures ▼

At least one image (poster or movie still) must be present in the card. Mark this if there is no image at all.

E6 Pictures Mismatch ▼

An image is present, but it clearly doesn't match the film — e.g., a different film's poster is shown.

editor tip
You don't need to verify the image is "official." Just check: does it visually appear to match the film (title on poster, recognisable scenes/characters)?
E7 No Title ▼

The card has no film/show title displayed. Title must always be present.

E8 Title Mismatch ▼

The title shown is incorrect — wrong film name, clearly wrong spelling, or a different title entirely.

E9 No Subtitle
KARINA CLARIFIEDUPDATED 5 MAY
▼

"Subtitle" here refers to language subtitles (e.g., English subtitles, Turkish subtitles) — not the film's title subtitle. Mark this when the user has requested subtitles but the card or player shows no subtitle availability.

editor tip
This is relevant when the user has specifically requested subtitles in their query (e.g., "English subtitle", "Turkish alt yazı"). Check whether the player or streaming links indicate subtitle availability.
Karina: only mark if the film has subtitles that are missing from the card. If subtitles simply don't exist for this title, this defect does not apply.
E10 Subtitle Mismatch
UPDATED 5 MAY
▼

The user requested a specific subtitle language, a subtitle is shown — but it's the wrong language. For example, user asked for English subtitles but only Turkish subtitles are available in the links.

E11 No Year of Release ▼

The release year is not displayed in the card.

where to find it
Metadata line below the pictures: e.g., 2016 · Adventure · 6+. The first number is the year. For TV series, a year range is acceptable (e.g., 2009–2020).
E12 Wrong Date ▼

The year is present but incorrect. Do a quick check if you suspect it's wrong.

where to find it

E14 Genre Mismatch ▼

The genre listed is clearly wrong — e.g., "comedy" for a horror film.

E15 No Age Rating ▼

No age rating is displayed in the card (e.g., 6+, 13+, 18+).

where to find it
Same metadata line: 2016 · Adventure · 6+. The last item is the age rating.
E16 Age Rating Issues
KARINA CLARIFIEDUPDATED 5 MAY
▼

The age rating shown is incorrect based on the Turkish rating standard.

benchmark = Turkish rating
Always judge age rating correctness by the Turkish rating — not the origin country's rating. This task serves Turkish market users, so the Turkish classification is what matters.

Example: Deadpool (2016) is R (≈17+) in the USA but 15+ in Turkey. If the card shows "15+" → correct, no defect. If it shows "17+" or "R" → mark Age Rating Issues.
fact-check required
A quick fact-check is needed to verify the Turkish age rating. Check a Turkish film database or the film's Turkish platform listing.
Karina: editors should judge based on the Turkish rating. A quick fact check is needed.
E17 No Review Score
KARINA CLARIFIEDUPDATED 5 MAY
▼

No score is shown in the card (e.g., IMDb 8.0). Mark this for both movies and TV shows/series — the requirement applies to all card types.

where to find it
Below the title — a yellow IMDb badge with a number, e.g., 8,0.
do not fact-check the score
Do NOT verify whether the score is accurate. Fact-checking is handled by a separate team. If a score is present, that's sufficient.
applies to series too
Even if most series cards in the task don't display a review score, still mark this defect when absent. The Wiki examples are illustrative only — not perfect samples.
Karina: "For now, let's mark it. These examples are used only to show which elements are typical for the responses, not to show any perfect samples in all terms." Also: "Review Score Invalid" has been removed from the Wiki — editors only check presence or absence, not accuracy.
E18 No Description ▼

The card has no descriptive text explaining what the film/show is about. A full description (plot or context) must be present.

short vs full description
The Short Description (a few adjectives: "Musical, bright, inspiring") is optional — don't mark it as missing. The Full Description (actual plot text) is required.
E19 No Duration
KARINA CLARIFIEDNEEDS CLARIFICATION
▼

The runtime is not shown anywhere in the card.

where to find it — two possible locations
1. Text below the description: e.g., "2h 34 min" — applies to ALL card types.
2. Timestamp on the embedded player: e.g., "1:35:47" in the corner of the video frame — only when there's a direct embedded player.
two card types behave differently
Card with embedded player: Duration may appear as a timestamp on the video frame. If it's there, no defect.

Card with external links only ("Şimdi İzle" / "Watch online" button): No video frame, so no timestamp. Duration must appear as text below the description. If it doesn't → mark this defect.
✗ Mark defect
Card shows "Şimdi İzle" button only. No "2h 34 min" text below the description.
✓ No defect
Card has "1:35:47" on the embedded player. OR "2h 34 min" written below the description.
Karina (via Elif): Duration requirement applies regardless of card type. Mark "No Duration" if absent — even for external-links-only cards.
Q12 still open: For TV shows/series, platforms typically display season/episode count rather than runtime. Should "No Duration" still be marked for series cards where duration is absent but season/episode count is present? Should "No season/episode count" be treated as an equivalent defect for series? Awaiting product team.

E21 No Player or Links Available
KARINA CLARIFIEDNEEDS CLARIFICATION
▼

Only applies to "To Watch" requests. When the user wants to watch a specific film, the card must include a working player or working streaming links. Mark this if they are missing or broken.

trigger words to identify "watch" intent
Look for: "izle", "watch", "online", "full HD", "where to watch", "streaming", platform names (Netflix, etc.), or dub/subtitle requests.
valid player forms (from screenshots)
✓ Embedded video player with play button and timestamp
✓ "Çevrimiçi izle" (Watch online) button with streaming icons
✓ "Watch online" dropdown with platform logos
✓ YouTube trailer link (trailers typically come from YouTube in this task)
Karina: (1) Any non-broken link counts — subscription requirement is irrelevant. (2) YouTube trailer links are valid. (3) Editors must click the link/player to verify it actually works.
Q11 still open: If links are from unofficial, unverified, or potentially pirated streaming sites — should editors mark a defect, or is any working link acceptable regardless of source legitimacy? Awaiting product team.
3 · Other
O1 Other
KARINA CLARIFIEDUPDATED 5 MAY
▼

Use this when you notice a significant, obvious flaw that clearly affects quality — but it doesn't fit any of the defects listed above.

use sparingly
Don't use "Other" for minor issues or personal preferences. Only mark when the flaw is substantial and clearly impacts usefulness.
known cases that go here
• User specifies a platform by name (e.g., "Avatar 2 Netflix") but model links to other valid platforms instead.
• User requests a specific number of films (e.g., "give me 10 movies") but the response returns fewer than requested.
Karina: if the user specifies a platform by name but the model gives links to other valid platforms → mark as "Other." If the user asks for 10 movies but only 8 are returned → also mark as "Other" for now.
Quick Reference · What to Fact-Check
general principle
Only fact-check when the Wiki explicitly requires it or when something is obviously wrong. Don't fact-check elements that are handled by a separate team (e.g., review scores).
